# claude-code-mac-notifications

> Native macOS notifications when Claude Code finishes a turn — with a 3-to-4 word AI summary and click-to-focus your specific terminal window.

When you start a long Claude Code task and tab away, this fires a notification the moment Claude is done. The body is a Title-Case action phrase generated by Haiku (e.g. *"Refactored Auth Module"*, *"Researched Game Server Hosting"*). Clicking the notification jumps you straight back to the exact Terminal/iTerm window/tab Claude was running in — not just the app generically.

## Features

- **Banner per turn** — fires on Claude Code's `Stop` event. Async, never blocks your prompt.
- **Banner when Claude is waiting on you** — fires on the `Notification` event (permission prompts, `AskUserQuestion`, etc.) with a sound and a "Needs your input" body. No LLM call, instant.
- **AI-summarized body for Stop** — a `claude -p --model haiku` call produces a 3-4 word action phrase from the last assistant message. Falls back to first-N-words if the LLM call fails.
- **Click-to-focus the right window** — captures the Claude session's TTY, then matches it via AppleScript on click. Works for **Apple Terminal** and **iTerm2**; other terminals fall back to whole-app activation.
- **Project-scoped title** — `Claude · <basename of cwd>` so you can tell sessions apart at a glance.
- **Dedup** — a fixed notification group means newer alerts displace older ones, so the Notification Center stays clean.
- **Claude-branded thumbnail** — the orange burst icon shows on the right of the banner.

## Requirements

| Tool                 | Why                                       |
|----------------------|-------------------------------------------|
| macOS                | Uses `osascript` and AppleScript          |
| [Homebrew](https://brew.sh) | Installs `terminal-notifier` for you |
| `jq`                 | JSON manipulation in scripts (`brew install jq`) |
| Claude Code CLI      | Provides the `claude` binary              |

> Notification permission for `terminal-notifier` is granted automatically the first time it fires, or via System Settings → Notifications.

## Installation

### Option 1 — script (no plugin marketplace)

```bash
git clone https://github.com/pramirez140/claude-code-mac-notifications.git
cd claude-code-mac-notifications
./install.sh
```

The installer is **idempotent** — re-run it any time to refresh the scripts. It:
1. Verifies macOS, `jq`, and the Claude CLI.
2. Installs `terminal-notifier` via brew if missing.
3. Copies hook scripts and the icon into `~/.claude/hooks/`.
4. Patches `~/.claude/settings.json` to register the `Stop` hook (replaces any prior entry pointing at `notify-stop.sh`).

### Option 2 — Claude Code plugin

```text
/plugin marketplace add pramirez140/claude-code-mac-notifications
/plugin install claude-code-mac-notifications@claude-code-mac-notifications
```

Restart Claude Code, or open `/hooks` once, so the watcher picks up the new Stop hook.

> The plugin format ships its own `terminal-notifier` dependency check at runtime — install it once with `brew install terminal-notifier` if the hook complains.

## Verifying it works

After install (or `/hooks` reload), end any Claude Code turn — you should see a banner about 5–10 seconds later. If nothing appears:

```bash
tail -20 ~/.claude/hooks/notify-stop.log
```

Common causes:
- **Do Not Disturb / Focus mode** is on. Banners are suppressed silently — toggle DND off in Control Center.
- **Notification permission for `terminal-notifier`** isn't granted. Open **System Settings → Notifications**, find `terminal-notifier`, set Alert Style to *Banners* or *Alerts*.
- **Hook didn't fire** — check the log; if there's no recent `fired:` line, the hook didn't run. Verify `~/.claude/settings.json` has the `Stop` entry.

## Customization

### Change the summary style

The summary is produced by a `claude -p --model haiku` call against this prompt (in `hooks/notify-stop.sh` near line 41):

```text
You will receive the last message from a coding assistant. Reply with a 3-to-4
word summary of what was just done. Action phrase only — no period, no quotes,
no preamble, no emoji. Title-Case the words.
```

Edit the prompt to suit your taste — funnier, lowercase, with emojis, in Spanish, whatever.

### Change the icon

Replace `~/.claude/hooks/claude-icon.png` (or the bundled `assets/claude-icon.png` for plugin installs) with any 256×256 PNG.

### Change the click target on unsupported terminals

The TTY-matching script lives at `hooks/focus-terminal.sh`. Apple Terminal and iTerm2 are wired up; for Ghostty / WezTerm / Warp / VSCode integrated terminal we just activate the whole app. PRs welcome.

### Disable temporarily

```text
/hooks
```

…and toggle the Stop hook off, or remove the entry from `~/.claude/settings.json`.

## How it works

```
┌─────────────────┐  Stop event   ┌──────────────────────┐
│  Claude Code    │ ────────────▶ │  notify-stop.sh      │
└─────────────────┘   (JSON       │  (this hook)         │
                       payload)   └──────────────────────┘
                                          │
                                          ├─ extract last assistant text from transcript JSONL
                                          ├─ claude -p --model haiku → 3-4 word summary
                                          ├─ resolve PARENT_TTY via ps ancestry walk
                                          └─ nohup terminal-notifier ... -execute focus-terminal.sh ... &

                            ┌──────────────────────┐  click   ┌──────────────────────┐
                            │   macOS notification │ ───────▶ │  focus-terminal.sh   │
                            └──────────────────────┘          │  AppleScript         │
                                                              │  match TTY → focus   │
                                                              └──────────────────────┘
```

The hook is `async: true` so it never blocks Claude's UI — terminal-notifier itself idles up to ~30 seconds waiting for the click. The fixed `-group "claude-stop"` means a new notification displaces the previous, so the count of background processes is bounded.

## Why so much engineering for a notification?

A few non-obvious lessons learned while building this:

1. **`-sender com.anthropic.claudefordesktop` looks reasonable but silently fails.** macOS routes notifications under the sender's bundle, and if Claude.app doesn't have notification permission, every notification disappears into the void. We post under terminal-notifier's own bundle and use `-contentImage` for the Claude logo instead.
2. **`-appIcon` is a no-op on macOS Tahoe.** Only `-contentImage` (the right-side thumbnail) is honored.
3. **The transcript JSONL has multiple "assistant" entries per turn** (text → tool_use → tool_result → text). Naïvely picking the last one with `tail -1` gives you a `tool_use` block with no text. We filter for entries containing a `text` content block.
4. **Embedded `\n` characters inside the assistant text break `tail -1`** when applied after `jq`. We normalize whitespace inside `jq` with `gsub("[\\n\\r\\t]+"; " ")` so the whole message is one logical line.
5. **`timeout` doesn't exist on stock macOS.** Use `perl -e 'alarm N; exec @ARGV'` for portability.
6. **`claude -p --model haiku` startup is dominated by MCP-server loading.** `--strict-mcp-config` cuts ~3s. `--no-session-persistence` stops one-shot summary calls from cluttering `~/.claude/projects/`.

## Uninstall

```bash
./uninstall.sh
```

Removes the Stop hook entry, deletes the scripts/icon. Leaves `terminal-notifier` installed (other tools may use it).
